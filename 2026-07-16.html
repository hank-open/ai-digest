PODCAST · ANTHROPIC 内部视角
Training Data
Anthropic's Katelyn Lesse & Angela Jiang: Building an Ecosystem, not a Walled Garden
核心观点：Anthropic Platform 团队正在从"执行层"向"协调层"迈进，未来的 alpha 不在于优化 prompt caching，而在于给 token 分配"职责"——这是 agent 工程的下一个量级。

Angela Jiang（Platform 负责人）和 Katelyn Lesse（高级工程师）来自 Anthropic 最核心的基础设施团队，她们同时负责内部产品的底层基础设施，以及对外的 API 和开发者平台。这个双内外视角让她们的洞察格外有说服力。

她们把整个平台抽象成三层蛋糕：

第一层：知识层（Knowledge） — 就是 Messages API、工具定义、MCP 等，告诉模型"怎么做事"。这层已经相对成熟。

第二层：执行层（Execution） — Claude Managed Agents 所在的层。解决的是长期运行的基础设施问题：sandbox 治理、session 存储、context 管理、prompt caching。Angela 说的很直接：模型现在已经非常"可引导"，很多老式 harness 里负责"引导模型走直线"的脚手架代码可以直接删掉了。

第三层：协调层（Coordination） — 她们花了最多时间讨论的地方。核心概念是 "strategies"（策略），也就是"给 token 分配职责"。同一个 token，你可以让它执行任务、让它回顾历史 agent session 并写入记忆、让它召唤更大的模型来"顾问"当前模型，或者让它在完成后作为"评分者"判断质量再决定是否重试。

"token 不是可互换的——你需要给它们不同的职责。这个 token 在建议，这个在执行，这个在做梦，这个在执行。你开始组合这些精心编排的策略，让它们协同工作。"

她们还分享了用户端的洞察：医疗行业有大量客户在用 computer use 桥接那些连 API 都没有的老系统；最激进的创新来自"AI native"的企业客户，而不是外部开发者；一个客户做了"把 MCP server 暴露在 agent 上层，让另一个 agent 把这个 agent 当工具调用"的架构，被她们称为"galaxy brain"方案，结果跑通了。

关于 token 成本优化（"tokenmaxxing → token rationalization"），她们的态度很清晰：不要靠设 cap 来限制 AI 使用，要靠设计更聪明的路由策略。困难任务给大模型，简单任务给便宜模型，这是未来 1-2 年企业部署 AI 的核心工程问题。Claude Tag 也在这次对话中被详细解读：它不只是一个 Slack bot，它是 Anthropic 对"org-level harness"的一次有主见的产品实验，核心是大量藏在水面下的 context engineering。
